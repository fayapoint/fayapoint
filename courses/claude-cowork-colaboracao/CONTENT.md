# MÓDULO 1: FUNDAMENTOS — Comunicação Eficaz

**MÓDULO 1: FUNDAMENTOS DO CLAUDE COWORK**
**AULA 2: COMUNICAÇÃO EFICAZ COM CLAUDE**

**Carga Horária:** 2 horas
**Indicadores Trabalhados:** 1. Comunicar tarefas eficientemente; 2. Iterar até o resultado ideal.

---

**CONTEÚDO PARA O ALUNO**

**1. Briefing Profissional**

Assim como briefaria um colega, inclua: contexto do projeto (o que é, para quem), objetivo específico (o que precisa ser entregue), formato desejado (como quer receber), referências (exemplos de qualidade), restrições (o que evitar, limitações), prazo/prioridade (urgente vs exploratório).

**2. O Poder do "Primeiro, Analise"**

Antes de pedir a entrega final, peça análise: "Antes de criar a proposta, analise o briefing e me diga: 1) Sua compreensão do pedido; 2) Pontos que precisa esclarecer; 3) Abordagem que pretende seguir." Isso evita retrabalho e alinha expectativas.

**3. Iteração Produtiva**

Não aceite a primeira versão. Itere como faria com um colega: "Ótima base. Agora: 1) Aprofunde a seção 3; 2) Adicione dados quantitativos; 3) Reduza a introdução pela metade; 4) Torne o tom mais assertivo na conclusão."

**4. Feedback Específico**

Ruim: "Não gostei, refaça." Bom: "A estrutura está boa mas: seção 2 está muito técnica para nosso público (gerentes não-técnicos), falta um executive summary no início, os números precisam de fonte, e o CTA final deve ser mais específico (agendar reunião, não 'entre em contato')."

# MÓDULO 2: ARTIFACTS — Documentos Interativos

**MÓDULO 2: ARTIFACTS E CRIAÇÃO**
**AULA 1: DOMINANDO ARTIFACTS**

**Carga Horária:** 3 horas
**Indicadores Trabalhados:** 1. Criar Artifacts de alta qualidade; 2. Usar diferentes tipos de Artifacts.

---

**CONTEÚDO PARA O ALUNO**

**1. O que são Artifacts**

Artifacts são entregas formatadas que Claude cria em painel separado: documentos, código, visualizações, SVGs, páginas web, diagramas Mermaid, componentes React. Diferente do texto na conversa, Artifacts são: editáveis, copiáveis como bloco, versionáveis (Claude pode iterar), exportáveis, renderizáveis (HTML/React executam no preview).

**2. Tipos de Artifacts**

- **Markdown:** Documentos formatados, relatórios, artigos
- **Code:** Python, JavaScript, HTML/CSS, SQL, qualquer linguagem
- **HTML:** Páginas web completas com CSS e JS
- **SVG:** Gráficos vetoriais, ícones, diagramas
- **React:** Componentes interativos com estado
- **Mermaid:** Fluxogramas, diagramas de sequência, gantt charts

**3. Pedindo Artifacts Eficazmente**

Diga explicitamente: "Crie um Artifact com..." ou "Gere como Artifact..." para garantir que Claude entregue no formato adequado ao invés de texto inline.

**Exemplo — Relatório:**
"Crie um Artifact Markdown com relatório mensal de vendas. Inclua: header com período e autor, executive summary, tabela de KPIs (meta vs realizado), análise por região, gráfico de tendência (como tabela ASCII), recomendações, próximos passos."

**4. Iterando Artifacts**

"Atualize o Artifact: mude o título para X, adicione seção Y após a introdução, reformate a tabela com colunas Z." Claude edita o Artifact existente mantendo o que já estava bom.

# MÓDULO 3: PROJECTS — Contexto Persistente

**MÓDULO 3: PROJECTS E CONTEXTO**
**AULA 1: DOMINANDO PROJECTS**

**Carga Horária:** 3 horas
**Indicadores Trabalhados:** 1. Configurar Projects eficazmente; 2. Usar Knowledge Base para contexto.

---

**CONTEÚDO PARA O ALUNO**

**1. O que são Projects**

Projects são espaços de trabalho no Claude com contexto persistente: Custom Instructions específicas do projeto, Knowledge Base (documentos uploadados como referência), Histórico de conversas organizadas, Compartilhamento com equipe (Team plan).

**2. Criando um Project Eficaz**

**Passo 1 — Custom Instructions:**
"Você é consultor de marketing digital para a empresa X. Nosso público-alvo é Y. Nossa marca tem tom Z. Sempre considere nosso budget de R$10.000/mês. Formate entregas como Artifacts quando possível."

**Passo 2 — Knowledge Base:**
Upload de documentos essenciais: brand guide, personas, histórico de campanhas, dados de performance, processos internos. Claude consulta estes documentos em TODAS as conversas do Project.

**Passo 3 — Conversas organizadas:**
Cada conversa dentro do Project já tem todo o contexto. "Crie proposta para cliente Z" → Claude já sabe sua empresa, marca, processos.

**3. Projects por Tipo de Trabalho**

- **Project: Cliente X** — Briefing + histórico + entregas
- **Project: Marketing Q1** — Planejamento + calendário + métricas
- **Project: Desenvolvimento** — Specs + codebase + docs
- **Project: Pesquisa** — Papers + notas + análises
- **Project: Gestão** — Processos + templates + reports

**4. Knowledge Base Strategy**

O que uplodar: documentos mais referenciados (brand guide, processos, templates), dados atualizados periodicamente (métricas, resultados), contexto que você repetiria em todo prompt (sobre a empresa, público, restrições). Limite: verifique o tamanho máximo do seu plano.

**CONTEÚDO PARA O ALUNO**

Claude pode processar ~150.000 palavras (~500 páginas) em uma única conversa. Isso significa: contratos inteiros, relatórios anuais completos, manuais técnicos, livros, múltiplos documentos simultaneamente. Nenhuma outra IA mainstream oferece essa capacidade com a mesma qualidade de análise.

**2. Framework de Análise de Documentos**

```
<document_analysis>
DOCUMENTO: [título/tipo]
OBJETIVO: [o que preciso extrair/entender]

ANALISE EM 5 CAMADAS:
1. RESUMO EXECUTIVO: Essência em 200 palavras

3. PONTOS-CHAVE: Os 10 insights mais importantes
4. ANÁLISE CRÍTICA: Pontos fortes, fracos, lacunas

</document_analysis>
```

**3. Contratos e Documentos Jurídicos**

Upload de contrato → "Analise este contrato como se eu fosse a parte contratante. Identifique: cláusulas favoráveis/desfavoráveis, riscos ocultos, pontos de negociação, obrigações e prazos, penalidades. Use linguagem simples, não juridiquês."

**5. Comparação de Documentos**

Upload de 2+ documentos → "Compare: diferenças em termos, condições, preços, prazos. Apresente em tabela. Recomende qual é mais vantajoso e por quê."

# MÓDULO 5: MCP — Model Context Protocol

**MÓDULO 5: MCP E INTEGRAÇÕES**
**AULA 1: ENTENDENDO E CONFIGURANDO MCP**

**Carga Horária:** 3 horas
**Indicadores Trabalhados:** 1. Entender o MCP; 2. Configurar integrações.

---

**CONTEÚDO PARA O ALUNO**

**1. O que é MCP**

O Model Context Protocol (MCP) é um padrão aberto criado pela Anthropic que permite ao Claude se conectar com ferramentas e dados externos. Pense em MCP como "USB para IA" — uma interface universal para conectar Claude a qualquer sistema.

**2. Como Funciona**

MCP Servers expõem "tools" (ações) e "resources" (dados) que Claude pode usar. Quando você configura um MCP server, Claude ganha a capacidade de: ler arquivos do seu computador, acessar APIs externas, executar código, consultar bancos de dados, interagir com serviços como GitHub, Slack, etc.

**3. MCP no Claude Desktop**

O Claude Desktop App suporta MCP nativamente. Configure em `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "ghp_xxx" }
    }
  }
}
```

**4. MCP Servers Populares**

- **Filesystem:** Ler/escrever arquivos locais
- **GitHub:** Repos, issues, PRs, code search
- **Slack:** Mensagens, canais, busca
- **PostgreSQL/SQLite:** Consultar bancos de dados
- **Brave Search:** Busca web
- **Puppeteer:** Automação de browser
- **Google Drive:** Acessar documentos

# MÓDULO 5: MCP — Integrações Avançadas

**MÓDULO 5: MCP E INTEGRAÇÕES**
**AULA 2: INTEGRAÇÕES AVANÇADAS E CUSTOM SERVERS**

**Carga Horária:** 2 horas
**Indicadores Trabalhados:** 1. Usar MCP para workflows complexos; 2. Criar MCP servers customizados.

---

**CONTEÚDO PARA O ALUNO**

**1. Workflow: Claude + GitHub**

Com MCP GitHub conectado: "Analise os PRs abertos no repo X. Para cada: resumo das mudanças, potenciais issues, sugestões de melhoria." Claude acessa diretamente o código, sem copy-paste.

**2. Workflow: Claude + Database**

Com MCP PostgreSQL: "Analise os dados de vendas do último trimestre. Quais produtos tiveram maior crescimento? Quais regiões performaram abaixo da meta?" Claude faz SQL queries diretamente.

**3. Workflow: Claude + Filesystem**

Com MCP Filesystem: "Leia todos os arquivos .md na pasta /docs e crie um índice organizado por categoria." Claude navega seus arquivos e processa sem upload manual.

**4. Criando MCP Server Custom**

Para integrações específicas da sua empresa:

```typescript
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";

const server = new McpServer({ name: "minha-empresa" });

server.tool("buscar_cliente", { cpf: "string" }, async ({ cpf }) => {
  const cliente = await db.query("SELECT * FROM clientes WHERE cpf = $1", [cpf]);
  return { content: [{ type: "text", text: JSON.stringify(cliente) }] };
});
```

Agora Claude pode buscar clientes do seu sistema com linguagem natural!
